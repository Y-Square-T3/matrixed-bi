#### **Openness**

leverage built-in data sources, chart types, and visual components. We believe a data visualization platform can, within
its system boundaries, offer extensibility across multiple layers. New extensions should be installable, updatable, and
removable in a plug-and-play fashion. Thus, Matrixed-BI aims to establish a standardized *
*`Open Data Visualization Platform`** framework, emphasizing standardization and openness in the following areas:

- ***Standardized Workflow***: Based on **Source â†’ View â†’ Chart â†’ Visualization**, Matrixed-BI implements a

- ***Standardized Interaction***: Visualizations support permission-controlled, standardized interactive features â€” such
  as filtering, drilling, cross-filtering, navigation, modals, sharing, downloads, scheduled delivery, etc.
- ***Standardized Plugin Architecture***: Standardized, pluggable extension interfaces or SDK specifications are

  demand.
